# Описание реализации ML-сервиса

## Выполненные требования

### ✅ (10 баллов) POST /forward
Реализован универсальный эндпоинт `/forward`, который поддерживает два формата:

1. **JSON запрос** (без изображений):
   - Принимает данные в теле запроса в формате JSON
   - Возвращает результаты в формате JSON с предсказанием и вероятностями

2. **multipart/form-data** (с изображением):
   - Принимает изображение через параметр `image`
   - Дополнительные параметры передаются в заголовках (`X-Asset`, `X-Window-Size`)
   - Возвращает изображение в формате base64 в JSON-документе

**Обработка ошибок:**
- Код 400 с текстом "bad request" для неверного формата запроса
- Код 403 с сообщением "модель не смогла обработать данные" при ошибках модели

### ✅ (5 баллов) GET /history
Реализован эндпоинт для получения истории всех запросов:
- Поддержка пагинации через параметры `skip` и `limit`
- История хранится в базе данных SQLite/PostgreSQL
- Возвращает общее количество записей и список элементов истории

### ✅ (2 балла, PRO) DELETE /history
Реализован эндпоинт для удаления истории:
- Требует JWT авторизации администратора
- Требует токен подтверждения в заголовке `X-Delete-Token`
- Удаляет все записи истории из базы данных

### ✅ (5 баллов, PRO) GET /stats
Реализован эндпоинт для статистики запросов:
- Среднее время обработки
- Квантили распределения (mean, 50%, 95%, 99%)
- Характеристики входных запросов (размеры входных данных)

### ✅ (3 балла, PRO) JWT авторизация
Реализована полноценная система авторизации:
- Создание пользователей (только администраторами)
- Получение JWT токена через `/token`
- Защита административных эндпоинтов через dependency injection
- Хеширование паролей с использованием bcrypt

### ✅ (5 баллов, PRO) Alembic миграции
Настроена система миграций базы данных:
- Начальная миграция создает таблицы `request_history` и `users`
- Конфигурация Alembic для работы с SQLite/PostgreSQL
- Поддержка автоматической генерации миграций

## Технологии

- **FastAPI** - веб-фреймворк для API
- **SQLAlchemy** - ORM для работы с БД
- **Alembic** - миграции базы данных
- **Pydantic** - валидация данных
- **PyTorch** - ML модель (LSTM)
- **JWT** - авторизация
- **bcrypt** - хеширование паролей

## База данных

Используется SQLite по умолчанию (можно переключить на PostgreSQL через переменную окружения `DATABASE_URL`).

**Таблицы:**
1. `request_history` - история всех запросов к API
2. `users` - пользователи системы (для авторизации)

## ML Модель

Реализована LSTM модель для бинарной классификации направления цены криптовалют:
- Вход: временное окно из 60 точек, каждая точка содержит 15 фичей
- Выход: предсказание класса (0 - падение, 1 - рост) и вероятности классов

Модель загружается из файла `models/model.pth` (если существует), иначе используется модель по умолчанию без обученных весов.

## Безопасность

- Пароли хешируются с использованием bcrypt
- JWT токены с настраиваемым временем жизни
- Токен подтверждения для критических операций (удаление истории)
- Валидация всех входных данных через Pydantic

## Запуск

1. Установить зависимости: `pip install -r requirements.txt`
2. Создать `.env` файл на основе `.env.example`
3. Выполнить миграции: `alembic upgrade head`
4. Создать администратора: `python scripts/create_admin.py --username admin --password your_password`
5. Запустить сервер: `uvicorn app.main:app --reload`
