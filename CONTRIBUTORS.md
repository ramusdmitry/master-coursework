# Распределение обязанностей в команде

## Команда: 2 участника

---

## Участник 1: Backend API и база данных

### Ответственность:

#### 1. Основной эндпоинт POST /forward
- Реализация универсального эндпоинта `/forward`:
  - Обработка JSON запросов (без изображений) - данные в теле запроса
  - Обработка multipart/form-data запросов (с изображением) - параметр `image`
  - Определение типа запроса по Content-Type заголовку
  - Обработка ошибок: код 400 для неверного формата, код 403 при ошибках модели
  - Интеграция с ML сервисом для получения предсказаний

#### 2. Модели базы данных (`app/models.py`)
- Модель `RequestHistory` для хранения истории всех запросов:
  - Поля: id, request_type, request_data, request_headers, response_data
  - processing_time_ms, success, error_message, input_size, created_at
- Модель `User` для системы авторизации:
  - Поля: id, username, hashed_password, is_admin, created_at

#### 3. Настройка базы данных (`app/database.py`)
- Конфигурация SQLAlchemy (engine, SessionLocal)
- Функция `get_db()` для dependency injection
- Поддержка SQLite и PostgreSQL через переменные окружения

#### 4. Схемы валидации (`app/schemas.py`)
- `ForwardRequestJSON` - схема для JSON запросов
- `ForwardResponseJSON` - схема ответа для JSON запросов
- `ForwardResponseImage` - схема ответа для запросов с изображением
- `HistoryItem`, `HistoryResponse` - схемы для истории
- `StatsResponse` - схема для статистики
- `UserCreate`, `UserResponse`, `Token` - схемы для авторизации

#### 5. Эндпоинт GET /history
- Получение истории всех запросов с пагинацией
- Параметры: `skip`, `limit`
- Возврат общего количества записей и списка элементов

#### 6. Эндпоинт DELETE /history (PRO)
- Удаление истории запросов
- Требует JWT авторизации администратора
- Требует токен подтверждения в заголовке `X-Delete-Token`
- Удаление всех записей из таблицы `request_history`

#### 7. Настройка Alembic и миграции (PRO)
- Конфигурация `alembic.ini`:
  - Настройка пути к скриптам миграций
  - URL базы данных из переменных окружения
- Конфигурация `alembic/env.py`:
  - Подключение моделей SQLAlchemy
  - Настройка для работы с SQLite и PostgreSQL
- Создание начальной миграции `001_initial_migration.py`:
  - Создание таблиц `request_history` и `users`
  - Создание индексов

#### 8. Интеграция в `app/main.py`
- Регистрация всех эндпоинтов в FastAPI приложении
- Настройка FastAPI приложения (title, description, version)
- Корневой эндпоинт `/` с описанием API
- Обработка ошибок и валидация на уровне приложения

### Файлы, за которые отвечает Участник 1:
- `app/main.py` (основная логика эндпоинтов)
- `app/models.py`
- `app/database.py`
- `app/schemas.py`
- `alembic.ini`
- `alembic/env.py`
- `alembic/versions/001_initial_migration.py`

## Участник 2: ML модель и авторизация

### Ответственность:

#### 1. ML модель и сервис (`app/ml_model.py`)
- Класс `SimpleLSTM`:
  - Архитектура LSTM модели для бинарной классификации
  - Параметры: input_size, hidden_size, num_layers, dropout
  - Метод `forward()` для предсказаний
- Класс `MLModelService`:
  - Загрузка обученной модели из файла `models/model.pth`
  - Загрузка scaler из файла `models/scaler.pkl`
  - Метод `predict()` для обработки JSON данных:
    - Валидация размера входных данных (минимум 60 временных точек)
    - Проверка количества фичей (15 для BTC)
    - Нормализация данных через scaler или стандартизацию
    - Выполнение предсказания через модель
    - Возврат предсказания и вероятностей классов
  - Метод `predict_image()` для обработки изображений:
    - Загрузка изображения через PIL
    - Конвертация в base64
    - Возврат метаданных (размеры, формат)
- Функция `get_ml_service()` для singleton паттерна

#### 2. JWT авторизация (`app/auth.py`)
- Функции для работы с паролями:
  - `verify_password()` - проверка пароля
  - `get_password_hash()` - хеширование пароля через bcrypt
- Функции для работы с JWT токенами:
  - `create_access_token()` - создание JWT токена
  - Декодирование токенов через python-jose
- Функции для аутентификации:
  - `authenticate_user()` - проверка username/password
  - `get_current_user()` - получение текущего пользователя из токена
  - `get_current_admin_user()` - проверка прав администратора
- Настройка OAuth2PasswordBearer для FastAPI

#### 3. Эндпоинт GET /stats (PRO)
- Расчет статистики запросов:
  - Общее количество запросов
  - Количество успешных/неуспешных запросов
  - Среднее время обработки
  - Квантили распределения (mean, 50%, 95%, 99%)
  - Статистика размеров входных данных (mean, min, max, median)

#### 4. Эндпоинт POST /token
- Получение JWT токена для авторизации
- Параметры: `username`, `password`
- Возврат access_token и token_type

#### 5. Эндпоинт POST /users
- Создание новых пользователей
- Требует авторизации администратора
- Валидация уникальности username
- Хеширование пароля перед сохранением

#### 6. Защита эндпоинтов
- Использование dependency injection для проверки прав доступа
- Защита административных эндпоинтов через `get_current_admin_user()`
- Интеграция авторизации в защищенные маршруты

#### 7. Вспомогательные скрипты
- `scripts/create_admin.py`:
  - Скрипт для создания первого администратора
  - Аргументы командной строки (username, password)
  - Создание таблиц БД при необходимости

#### 8. Документация
- `README_SERVICE.md`:
  - Описание всех эндпоинтов API
  - Примеры использования (curl команды)
  - Инструкции по установке и запуску
  - Описание структуры проекта
- `IMPLEMENTATION.md`:
  - Описание реализации всех требований
  - Технологический стек
  - Особенности реализации

### Файлы, за которые отвечает Участник 2:
- `app/ml_model.py`
- `app/auth.py`

- `README_SERVICE.md`
